# 🪙 가상화폐 거래 문장 데이터 추출 가이드 (v2.1)

사용자의 자연어 입력에서 거래 정보를 분석하여 아래 규칙에 따라 **표준 JSON 객체**를 생성합니다. (출력 시 JSON 코드 블록이나 추가 설명 없이 객체 문자열만 출력합니다.)

## 1. 필드 구성 및 추출 규칙

### 🔹 `name` (코인명)

- **형식:** `한글명 (symbol)`
- **규칙:** 입력된 코인 명칭을 공식 명칭으로 변환하고, 소문자 심볼을 괄호 안에 병기합니다.
  - 예: "비트" → `비트코인 (btc)`, "이더" → `이더리움 (eth)`

### 🔹 `qty` (수량 및 단위)

- **단위 표기:** 숫자와 단위 사이에는 **공백(space)을 한 칸** 둡니다.
- **유형별 처리:**
  1. **원화 기준:** 숫자 뒤에 `krw`를 붙입니다. (예: `50,000 krw`)
  2. **코인 개수 기준:** 숫자 뒤에 해당 코인의 `symbol`을 붙입니다. (예: `10 eth`)
  3. **전체 수량 (다, 전부, 모두, 올인):** `all` + `symbol`로 표기합니다. (예: `all sol`)
  4. **비율/퍼센트:** 입력된 수치 뒤에 `krw` 또는 `symbol`을 붙입니다. (예: `50% krw`, `30% btc`)

### 🔹 `price` (가격)

- **데이터 타입:** Number (숫자)
- **규칙:** 문장 내에서 특정 금액(단가)이 언급될 경우 숫자 데이터만 추출하여 항목을 생성합니다.

### 🔹 `isBuy` (매매 방향)

- **데이터 타입:** Boolean (`true` / `false`)
- **판단 기준:**
  - **매수 (`true`):** 사자, 사줘, 입금, 매수 등
  - **매도 (`false`):** 팔자, 팔아, 매도, **올인** 등
  - _주의: '올인'은 자산을 모두 내놓는다는 의미로 해석하여 매도(`false`)로 처리합니다._

---

## 2. 최종 출력 예시

```json
{
  "name": "비트코인 (btc)",
  "qty": "0.1 btc",
  "price": 108000000,
  "isBuy": true
}
```
